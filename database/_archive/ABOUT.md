# database/_archive

Исторические/одноразовые SQL (диагностика, разовые правки, ранние миграции), вынесены
из `database/` для чистоты. Источник правды схемы прода — `database/migrations/`
(датированные, применяются вручную через Management API). Эти файлы — только история/
референс, в сборке и приложении не используются. (README.md в репо игнорируется —
поэтому файл называется ABOUT.md.)
