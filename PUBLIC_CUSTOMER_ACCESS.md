# Система публичного доступа к профилю клиента

## Обзор

Реализована система для предоставления клиентам доступа к их личной информации без необходимости авторизации в системе.

## Что реализовано

### 1. Страница профиля клиента (/customer/:id)
**Файл:** `src/pages/CustomerProfile.tsx`

**Доступ:** Только для авторизованных пользователей СТО

**Функционал:**
- Полная информация о клиенте (имя, телефон, email, адрес, заметки)
- Список всех автомобилей клиента
- История всех заявок с фильтрацией по статусу
- Кнопка "Назад к клиентам"

### 2. Публичная страница клиента (/public/customer/:id)
**Файл:** `src/pages/PublicCustomerView.tsx`

**Доступ:** Публичный (без авторизации)

**Функционал:**
- Имя клиента скрыто (отображается как "******")
- Список автомобилей клиента с полной информацией
- История обслуживания с детальной информацией о заявках
- Информация о статусе оплаты (запчасти/работы)
- Красивый дизайн с акцентом на читаемость

### 3. Модификация страницы "Клиенты"
**Файл:** `src/pages/Customers.tsx`

**Изменения:**
- ✅ Строки таблицы НЕ кликабельны
- ✅ Имя клиента — кликабельная ссылка → открывает `/customer/:id`
- ✅ Новая кнопка "Публичная ссылка" (иконка Link) в столбце действий
- ✅ При клике на кнопку — ссылка копируется в буфер обмена
- ✅ Toast уведомление об успешном копировании

### 4. Маршрутизация
**Файл:** `src/App.tsx`

Добавлены маршруты:
- `/customer/:id` — защищенный профиль клиента (для сотрудников СТО)
- `/public/customer/:id` — публичная страница (без авторизации)

## Использование

### Для сотрудников СТО:

1. Перейти на страницу "Клиенты"
2. Нажать на имя клиента → откроется детальный профиль
3. Нажать на иконку Link → публичная ссылка скопируется в буфер

### Для клиентов:

1. Получить ссылку от СТО (формат: `https://ваш-домен/public/customer/UUID`)
2. Перейти по ссылке без авторизации
3. Просмотреть свои автомобили и историю заявок

## Безопасность

### Публичная страница:
- ✅ Не требует авторизации
- ✅ Имя клиента скрыто (******)
- ✅ Показывается только информация о заявках и автомобилях
- ✅ Нет доступа к персональным данным (телефон, email, адрес)
- ✅ RLS политики разрешают только чтение (SELECT)

### Защищенная страница:
- ✅ Требует авторизации
- ✅ Доступна только сотрудникам СТО
- ✅ Показывает полную информацию о клиенте

### Настройка публичного доступа:
**ВАЖНО:** Для работы публичных ссылок необходимо выполнить SQL миграцию:

1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Выполните скрипт из файла `database/migrations/add_public_access_policies.sql`

Это создаст RLS политики, разрешающие анонимный доступ на чтение для:
- `appointments` - заявки СТО
- `vehicles` - автомобили
- `customers` - клиенты СТО (только phone для связи)
- `parts_orders` - заказы запчастей
- `parts_order_items` - позиции заказов
- `parts_inventory` - информация о запчастях
- `parts_customers` - клиенты разборки

## Дополнительные улучшения (опционально)

### Рекомендуемые:

1. **Система токенов доступа**
   - Вместо прямого ID использовать случайный токен
   - Таблица `customer_access_tokens` с полями:
     - `id` (UUID)
     - `customer_id` (UUID)
     - `token` (уникальная строка)
     - `created_at`
     - `expires_at` (опционально)

2. **Ограничение доступа по времени**
   - Возможность задать срок действия ссылки
   - Автоматическая деактивация устаревших токенов

3. **Аналитика**
   - Отслеживание количества просмотров публичной страницы
   - Логирование доступа к публичным ссылкам

4. **QR-коды**
   - Генерация QR-кода для публичной ссылки
   - Удобно для печати на квитанциях/документах
