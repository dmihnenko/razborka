# Модуль "Мои автомобили"

Система учета личных автомобилей, купленных на аукционах США, с детальным учетом расходов, галереей фотографий и публичным доступом по коду.

## ✅ РЕАЛИЗОВАНО

### 1. База данных
- ✅ Таблица `personal_vehicles` - хранение автомобилей
- ✅ Таблица `vehicle_share_links` - коды доступа
- ✅ RLS политики для безопасности
- ✅ Публичный просмотр по коду (без авторизации)
- ✅ Триггер auto-update для `updated_at`

### 2. Структура данных
- ✅ TypeScript типы (`src/types/personalVehicles.ts`)
- ✅ 4 категории расходов: Авто, Запчасти, Работа, Доп.расходы
- ✅ 3 фото-альбома: USA, Port, Arrival
- ✅ Валюты: USD, UAH с авто-конвертацией
- ✅ Состояние запчастей: новая/б/у/неоригинал

### 3. API Service
- ✅ CRUD операции для автомобилей
- ✅ Управление расходами по категориям
- ✅ Загрузка/удаление фото в альбомы
- ✅ Генерация уникальных 4-значных кодов
- ✅ Валидация кодов доступа
- ✅ Отметка "продан" с расчетом прибыли

### 4. Компоненты
- ✅ `ExpenseModal` - добавление/редактирование расхода
- ✅ `ExpenseCategory` - отображение категории расходов
- ✅ `PersonalVehicleExpenses` - управление всеми расходами
- ✅ `VehicleGallery` - галерея с drag & drop
- ✅ `ShareLinkModal` - генерация кодов доступа
- ✅ `PersonalVehicleModal` - создание автомобиля

### 5. Страницы
- ✅ `MyVehicles` - список активных автомобилей
- ✅ `MyVehiclesArchive` - архив проданных
- ✅ `PublicPersonalVehicleView` - детальная страница
- ✅ `VehicleAccessPage` - ввод кода доступа

### 6. Функционал
- ✅ Загрузка фото через imgBB API
- ✅ Drag & Drop для фото
- ✅ Валидация файлов (тип, размер)
- ✅ Курс USD для конвертации UAH
- ✅ Автоматический расчет total_cost
- ✅ Навигация клавиатурой в галерее (ESC, ← →)
- ✅ Адаптивная верстка (mobile, tablet, desktop)
- ✅ Расчет прибыли/убытка при продаже

## 🚀 ИСПОЛЬЗОВАНИЕ

### Добавление автомобиля
1. Перейти на `/my-vehicles`
2. Нажать "Добавить"
3. Заполнить форму: марка, год, VIN, курс USD
4. Загрузить фото
5. Сохранить

### Учет расходов
1. Открыть автомобиль
2. Установить курс USD (если еще не установлен)
3. В каждой категории нажать "Добавить"
4. Заполнить название, сумму, валюту
5. Для запчастей - указать состояние
6. Автоматический расчет Grand Total

### Загрузка фотографий
1. Выбрать альбом (USA / Port / Arrival)
2. Перетащить фото или нажать для выбора
3. Поддержка: JPEG, PNG, GIF, WebP (макс 10MB)
4. Клик по фото → просмотр в модалке
5. Навигация: ← → или кнопки Prev/Next
6. ESC для закрытия

### Поделиться доступом
1. Нажать "Поделиться"
2. Указать срок действия (опционально)
3. Сгенерировать код (4 цифры)
4. Скопировать код или ссылку
5. Отправить клиенту

### Клиент: Просмотр по коду
1. Перейти на `/vehicle-access`
2. Ввести 4-значный код
3. Получить доступ к автомобилю (только чтение)
4. Может просматривать расходы и фото

### Продажа автомобиля
1. Открыть автомобиль
2. Нажать "Отметить как проданный"
3. Указать цену продажи
4. Автомобиль переходит в архив
5. Отображается прибыль/убыток

## 🔒 БЕЗОПАСНОСТЬ

### RLS Политики
- ✅ Пользователи видят только свои автомобили
- ✅ Публичный просмотр ПО КОДУ (без авторизации)
- ✅ Только владельцы могут изменять/удалять
- ✅ Коды доступа видны всем (для валидации)
- ✅ Создавать коды могут только владельцы

### Валидация
- Марка и модель: 3-100 символов
- Год: 1900-2027
- VIN: 17 символов или пусто
- Курс USD: > 0
- Код доступа: ровно 4 цифры

## 📊 РАСЧЕТЫ

### Общая стоимость (total_cost)
```
totalCost = Σ(lotItems) + Σ(partsItems) + Σ(workItems) + Σ(additionalItems)

Где каждый item:
- USD: берется как есть
- UAH: делится на usdRate
```

### Прибыль при продаже
```
profit = salePrice - totalCost

> 0: зеленым
< 0: красным
```

## 🔄 WORKFLOW

### Типичный сценарий
1. Пользователь покупает авто на Copart
2. Добавляет в "Мои авто" с фото из США
3. Устанавливает курс USD
4. Добавляет расходы: лот $5000, доставка $1200
5. Прибытие → добавляет фото в Port
6. Ремонт → добавляет запчасти и работы
7. Готовое авто → фото в Arrival
8. Генерирует код доступа для клиента
9. Клиент просматривает все расходы и фото
10. Продает → отмечает как проданный → видит прибыль

## 🐛 TROUBLESHOOTING

### Не загружаются фото
- Проверить VITE_IMGBB_API_KEY в .env
- Размер файла < 10MB
- Формат: JPEG, PNG, GIF, WebP

### Ошибка доступа
- Проверить выполнение миграции personal_vehicles.sql
- Проверить RLS политики в Supabase

### Не работает код доступа
- Код должен быть активен (is_active = true)
- Проверить срок действия (expires_at)
- Код ровно 4 цифры
