2. Скопируйте весь код из файла `database/fix_sto_access_and_assignments.sql`
3. Вставьте в SQL редактор и нажмите **Run**

### Шаг 2: Обновите существующие данные (опционально)
Если у вас уже есть автомобили и заявки, обновите их `sto_company_id`:

```sql
-- Обновить автомобили, присвоив им sto_company_id из их клиентов
UPDATE vehicles v
SET sto_company_id = c.sto_company_id
FROM customers c
WHERE v.customer_id = c.id
AND v.sto_company_id IS NULL;

-- Проверка
SELECT 
  v.id, 
  v.brand, 
  v.model,
  v.sto_company_id,
  sc.name as company_name
FROM vehicles v
LEFT JOIN sto_companies sc ON v.sto_company_id = sc.id;
```

## Как это работает

### Клиенты и автомобили
- Все работники СТО видят **всех клиентов** своей СТО
- Все работники СТО видят **все автомобили** своей СТО
- При создании автомобиля автоматически привязывается `sto_company_id` из профиля пользователя

### Заявки (Appointments)
**Владелец СТО:**
- ✅ Видит все заявки своей СТО
- ✅ Может создавать заявки
- ✅ Может назначать заявки работникам
- ✅ Может редактировать и удалять любые заявки
- ✅ Видит статистику: количество заявок в работе

**Работник СТО:**
- ✅ Видит только те заявки, которые:
  - Он сам создал (`created_by = user_id`)
  - Ему назначены (`assigned_to = user_id`)
- ✅ Может создавать новые заявки
- ✅ Может редактировать только свои заявки
- ❌ Не может удалять заявки

### Назначение работников
1. При создании заявки на шаге "Проверка" появляется выпадающий список работников
2. Владелец может выбрать работника из списка
3. После создания работник сразу увидит эту заявку в своем списке
4. В таблице заявок отображается, кому назначена каждая заявка

## Тестирование

### 1. Как владелец СТО
- Зайдите под учетной записью с ролью `sto_owner`
- Проверьте Dashboard - должна быть статистика по вашей СТО
- Создайте заявку и назначьте работнику
- Проверьте, что видите все заявки

### 2. Как работник СТО
- Зайдите под учетной записью с ролью `sto_worker`
- Проверьте, что видите только:
  - Заявки, которые вы создали
  - Заявки, назначенные вам владельцем
- Попробуйте создать свою заявку

### 3. Проверка изоляции данных
- Создайте две разные СТО
- Убедитесь, что работники одной СТО не видят данные другой СТО

## Примечания
- RLS политики автоматически обеспечивают изоляцию данных
- Все запросы фильтруются на уровне базы данных
- Админы имеют полный доступ ко всем данным
