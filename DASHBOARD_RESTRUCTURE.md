# Сводка изменений - Реструктуризация Dashboard и История

## Выполненные изменения

### 1. ✅ Создана система логирования изменений

**Файл:** `database/migrations/create_activity_log.sql`

- Таблица `activity_logs` для хранения всех изменений
- Автоматическое логирование через триггеры:
  - Заявки (appointments): создание, обновление, изменение статуса, назначение работника, оплата, архивирование
  - Клиенты (customers): создание, обновление, удаление
- Функция автоматического удаления логов старше 60 дней
- RLS политики для безопасности
- Индексы для быстрого поиска

**Что логируется:**
- Кто внес изменения (имя, email)
- Когда произошло изменение
- Что изменилось (описание + старое/новое значение в JSON)
- Тип действия (создано, обновлено, удалено, статус изменен, назначено, оплата)

## Инструкции по установке

### Шаг 1: Создать таблицу и триггеры

1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Создайте новый запрос
4. Скопируйте содержимое файла `database/migrations/create_activity_log.sql`
5. Вставьте и выполните (Run)

### Шаг 2: Включить pg_cron

1. В Supabase Dashboard перейдите: **Database** → **Extensions**
2. Найдите **pg_cron** в списке
3. Нажмите **Enable**

### Шаг 3: Настроить автоматическое удаление

В SQL Editor выполните:

```sql
SELECT cron.schedule(
  'delete-old-activity-logs',
  '0 2 * * *',  -- Каждый день в 2:00 ночи
  $$SELECT delete_old_activity_logs()$$
);
```

### Шаг 4: Проверить работу

```sql
-- Посмотреть запланированные задачи
SELECT * FROM cron.job;

-- Посмотреть последние логи
SELECT * FROM activity_logs ORDER BY created_at DESC LIMIT 10;
```

## Что изменится для пользователей

### Владельцы СТО
- Главная страница теперь показывает все заявки сразу (не нужно переходить в раздел "Записи")
- Новая кнопка **История** для просмотра всех изменений
- Можно отслеживать, кто и что изменял в системе

### Работники
- Видят только свои заявки на главной странице
- История недоступна (только для владельцев)

### Клиенты (роль user)
- Остаются на странице "Мои автомобили" (ничего не изменилось)

## Технические детали

### Безопасность
- RLS политики: каждая СТО видит только свои логи
- Логи нельзя изменить или удалить вручную
- Удаление только автоматическое через cron

### Производительность
- Индексы на всех ключевых полях (sto_company_id, user_id, entity_type, created_at)
- LIMIT 100 в UI для быстрой загрузки
- JSONB для эффективного хранения старых/новых значений

### Хранение данных
- Логи: хранятся 60 дней
- Заявки, клиенты, автомобили: **НЕ затрагиваются**, остаются навсегда

## Поддержка

Если возникнут проблемы:
1. Проверьте логи выполнения cron: `SELECT * FROM cron.job_run_details ORDER BY start_time DESC LIMIT 10;`
2. Проверьте, создались ли триггеры: `SELECT * FROM pg_trigger WHERE tgname LIKE 'trigger_log_%';`
