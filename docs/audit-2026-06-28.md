Комплексный аудит силами специалистов (SEO, безопасность, производительность, функционал/рост, подписки) + тимлид-синтез. Severity: **P0** критично · **P1** важно · **P2** улучшение.

## 0. Резюме (тимлид)

Проект в хорошей форме: прошёл волну hardening (RLS на активных таблицах, Edge Functions проверяют права), есть edge-SEO, lazy-роуты, нормальный QueryClient. Критичные зоны — **монетизация (подписки enforced только на клиенте, оплаты нет)** и **один захардкоженный секрет**. Остальное — приоритизированные улучшения.

**Порядок работ:** P0 безопасность → P0/P1 подписки (server enforcement) → P1 perf quick wins → P1 SEO → P1 функционал (резерв склада, уведомления) → P2 рост.

## 2. Подписки (монетизация) — ключевой блок

**Как сейчас:** планы в БД `subscriptions` (Старт 400 / Бизнес 600 / Профи 800 ₴ + Демо), активный план тенанта — в `company_subscriptions` (`is_active`, `end_date`). Активация **ручная** (заявка → админ `approveSubscriptionRequest`). Оплаты онлайн нет.

| Sev | Пробел | Где |
|-----|--------|-----|
| **P0** | Лимиты (товары/авто/сотрудники) enforced ТОЛЬКО на клиенте (`useSubscriptionLimits.canCreate`). `insert` мимо UI обходит лимит. Нет триггеров/RLS. | `partsService.ts:42/455`, `useSubscription.ts:140` |
| **P0** | Истечение не enforced на сервере: `end_date` прошёл → строка остаётся `is_active=true`, товары видны в маркете. Скрытие только на клиенте. | `useSubscription.ts:39` |
| **P0** | Нет онлайн-оплаты (всё вручную). | `PartsSubscriptionPage.tsx:371` |
| **P1** | Лимит сотрудников не применяется вообще (даже на клиенте) — `create-user` без проверки. | `Users.tsx`, `functions/create-user` |
| **P1** | Нет статуса (trial/active/past_due/expired), trial, грейс-периода; нет истории платежей (`assignSubscription` затирает строку). | `subscriptionService.ts:105` |
| **P2** | Фото-лимит хардкод `MAX_PHOTOS=5`, не привязан к тарифу. | `PartsInventory.tsx:32` |

**План (реализуется):**
1. Серверный enforcement — триггеры BEFORE INSERT на `parts_inventory`/`parts_vehicles` по активному плану (зеркало клиентских лимитов, демо-фолбэк 50/3).
2. Лимит сотрудников в Edge `create-user`.
3. Статус/срок: колонки `subscription_status`+`current_period_end` (кэш на `parts_companies`), cron-деактивация по сроку (рядом с курсом USD в worker).
4. История платежей: таблица `subscription_payments` (append-only).
5. Онлайн-оплата: **LiqPay** (ПриватБанк, ФОП-friendly, рекуррент) — отдельный этап с ключами мерчанта; ручной режим оставить fallback.
