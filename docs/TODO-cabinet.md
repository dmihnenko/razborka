### Расширенная модель статусов + авто-delivered (нужен sign-off)
Текущая модель — `new/in_progress/completed/cancelled` (CHECK-constraint). Расширение до `new→assembling→ready→shipped→delivered→completed` = миграция enum + переработка воркфлоу заказа + миграция существующих данных. ⚠️ Поведенческое изменение на живой системе — согласовать процесс с владельцем. Авто-`delivered` из НП (коды 9/10/11, хелперы `shipDelivered` уже есть) имеет смысл только после появления отдельного статуса `delivered`.
