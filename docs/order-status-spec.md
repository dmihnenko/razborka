## Текущее состояние (как есть)
- Статусы: `new` · `in_progress` · `completed` · `cancelled` (CHECK-constraint на `parts_orders.status`).
- Канбан-доска заказов уже есть (3 колонки, drag-and-drop) — `PartsOrders.tsx`.
- Таймлайн заказа есть (из `parts_activity_log`) — `PartsOrderDetails.tsx`.
- ТТН создаётся вручную → регистрируется посылка (`createShipment`), Telegram-уведомление при отправке.
- Трекинг НП: коды `9/10/11` = доставлено/получено, `14/102/103/108` = проблема/возврат.
- Инвентарь: при `completed` позиции должны становиться `sold`; при `cancelled`/возврате → `available`.

## Принятая модель (4 этапа + отмена)

> Правка от пользователя: убраны «Готов» и «Доставлен».
> Доставка НП (трекинг 9/10/11) остаётся в разделе «Доставка», но статус заказа НЕ меняет.

| # | Статус (код) | Название | Что значит | Инвентарь |
|---|---|---|---|---|
| 1 | `new` | Новый | Заказ создан, позиции добавляются | — |
| 2 | `assembling` | Сборка | Подбираем/комплектуем заказ | позиции → **резерв** |
| 3 | `shipped` | Отправлен | Создана ТТН / передан в доставку | резерв |
| 4 | `completed` | Завершён | Деньги получены, сделка закрыта | позиции → **продано** |
| ✕ | `cancelled` | Отменён | Отмена с любого этапа | позиции → **в наличие** |

## Авто-правила переходов
- **new → assembling** — при добавлении первой позиции (или вручную). **Авторезерв** всех позиций заказа.
- **assembling → shipped** — **авто** при создании ТТН (уже есть хук `onTtnCreated`).
- **shipped → completed** — вручную («Деньги получены»). Позиции → продано.
- **assembling → completed** — для самовывоза (минуя «Отправлен»), см. вопрос 2.
- **любой → cancelled** — вручную; возврат позиций в наличие.

Авто-`delivered` из НП убран вместе со статусом «Доставлен» — поведенческой магии меньше, доставка отслеживается отдельно в разделе «Доставка».

## Эффекты на инвентарь (когда позиция меняет статус)
- **Резерв** — на `assembling` (и держится до завершения/отмены). Нужно поле учёта (статус позиции `reserved`).
- **Продано** — на `completed` (фиксируем `sold_price`, `sold_at`, `exchange_rate_at_sale`).
- **В наличие** — на `cancelled` и при возврате (sold/reserved → available).

## Открытые вопросы (ответь / поправь)
1. **Самовывоз / продажа без доставки**: нужен ли путь `Сборка → Завершён` напрямую (минуя «Отправлен»)? *(Предлагаю — да: кнопка «Завершить» доступна и из «Сборки», и из «Отправлен».)*
2. **Авторезерв**: резервировать позиции на «Сборке» (как предложено) или сразу при добавлении позиции в «Новый»?
3. **Названия** статусов — устраивают? (Новый / Сборка / Отправлен / Завершён / Отменён.)
4. **Маппинг старых данных**: текущий `in_progress` → `assembling` (Сборка). Ок?

## Риски
- Поведенческое изменение на живых заказах — миграция существующих статусов делается осторожно, без потери данных.
- Авто-`delivered`/`completed` — не должны срабатывать неожиданно; начинаем с минимума авто-правил, остальное вручную.
- Резерв требует корректного снятия при отмене/возврате/удалении заказа.
