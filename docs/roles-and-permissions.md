# Система ролей и прав доступа

## Роли пользователей

### 1. Администратор (admin)
- **Описание**: Полный доступ ко всем функциям системы
- **Права**:
  - Управление всеми модулями
  - Управление пользователями и их ролями
  - Доступ ко всем отчетам и аналитике
  - Настройка системы

### 2. Пользователь (user)
- **Описание**: Базовый уровень доступа
- **Права**:
  - Просмотр общедоступной информации
  - Ограниченный доступ к функциям

### 3. Владелец СТО (sto_owner)
- **Описание**: Владелец станции технического обслуживания
- **Права**:
  - Полное управление СТО
  - Доступ ко всем отчетам и аналитике СТО
  - Управление персоналом СТО
  - Просмотр финансовой информации
  - Настройка услуг и прайс-листа

### 4. Работник СТО (sto_worker)
- **Описание**: Сотрудник станции технического обслуживания
- **Права**:
  - Создание и редактирование заказ-нарядов
  - Работа с клиентами и автомобилями
  - Просмотр и использование услуг
  - Работа с запчастями (использование в работе)
  - Просмотр назначенных задач

### 5. Владелец разборки (parts_owner)
- **Описание**: Владелец авторазборки (будущий функционал)
- **Права**:
  - Полное управление авторазборкой
  - Доступ к отчетам по разборке
  - Управление персоналом разборки
  - Ценообразование на запчасти
  - Финансовая аналитика

### 6. Работник разборки (parts_worker)
- **Описание**: Сотрудник авторазборки (будущий функционал)
- **Права**:
  - Управление инвентарем запчастей
  - Оформление продаж
  - Работа с заказами
  - Приемка автомобилей на разборку

### 7. Владелец автомагазина (store_owner)
- **Описание**: Владелец автомагазина (будущий функционал)
- **Права**:
  - Полное управление автомагазином
  - Доступ к отчетам и аналитике
  - Управление персоналом магазина
  - Управление ассортиментом
  - Финансовая отчетность

### 8. Работник автомагазина (store_worker)
- **Описание**: Сотрудник автомагазина (будущий функционал)
- **Права**:
  - Управление инвентарем товаров
  - Оформление продаж
  - Работа с клиентами
  - Обработка заказов

## Структура прав доступа (permissions)

Права хранятся в формате JSON и имеют следующую структуру:

```json
{
  "all": true,  // Полный доступ (только для admin)
  "read": true, // Право на чтение
  "sto": {      // Права для СТО
    "manage": true,       // Полное управление
    "reports": true,      // Доступ к отчетам
    "staff": true,        // Управление персоналом
    "work_orders": true,  // Работа с заказ-нарядами
    "customers": true,    // Работа с клиентами
    "vehicles": true      // Работа с автомобилями
  },
  "parts_shop": {  // Права для авторазборки
    "manage": true,
    "reports": true,
    "staff": true,
    "inventory": true,
    "sales": true
  },
  "auto_store": {  // Права для автомагазина
    "manage": true,
    "reports": true,
    "staff": true,
    "inventory": true,
    "sales": true
  }
}
```

## Применение схемы

1. **Применить основную схему**:
   - Откройте Supabase Dashboard: https://supabase.com/dashboard/project/hwckvddevjucuzxdoqqh/sql/new
   - Скопируйте и выполните содержимое файла `database/schema.sql`

2. **Применить триггер для профилей**:
   - В том же SQL Editor выполните содержимое файла `database/user_profile_trigger.sql`

3. **Проверка**:
   - После выполнения будут созданы таблицы `roles` и `user_profiles`
   - В таблице `roles` будут добавлены 8 ролей
   - При регистрации новых пользователей автоматически создаются профили

## Назначение ролей

Роли назначаются через страницу "Пользователи" в CRM:
1. Перейдите в раздел "Пользователи"
2. Нажмите на иконку шестеренки напротив пользователя
3. Выберите нужную роль из списка
4. Сохраните изменения

## Безопасность

- Все таблицы защищены Row Level Security (RLS)
- Управление ролями доступно только администраторам
- Пользователи могут видеть только свою информацию (кроме админов)
- При удалении пользователя из auth.users автоматически удаляется его профиль
