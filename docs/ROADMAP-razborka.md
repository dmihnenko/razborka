# РОАДМАП — AutoRozborka (SaaS для авторазборок)

> Сводный приоритизированный план развития продукта. Составлен на основе аудита кода (июнь 2026) и четырёх стратегических направлений: функционал/модули, UX владельцев, UX покупателей маркетплейса, подписки.
> Шкала усилий: **S** ≤ 1 недели · **M** = 2–4 недели · **L** = 1,5+ мес. Команда небольшая, весь бэкенд — Supabase (Postgres + RLS + Edge Functions), без отдельного сервера.

---

## 1. Видение и позиционирование

Мы не конкурируем с OLX и авто.ria за объявления — мы становимся **операционной системой разборки**: владелец один раз ведёт склад у нас (авто-донор → запчасти с фото и местом хранения → продажа), а продукт сам (а) **считает окупаемость каждой машины** — главный вопрос бизнеса «отбил ли я это авто и какие брать дальше», который не отвечает ни Excel, ни OLX; (б) **выгружает склад во все каналы UA-рынка** (фиды OLX/авто.ria, Telegram-канал, собственный маркетплейс /market) и принимает заявки покупателей с мгновенным уведомлением в Telegram; (в) **закрывает сделку** — заявка с маркета в один клик становится заказом, отправка — наложенным платежом Новой Почты. Формула рынку: *«Веди склад один раз — продавай везде — и знай, какие машины покупать»*. Монетизация — подписочная лестница, где за 400₴ платят за заявки с маркета, за 600₴ — за каналы продаж, за 800₴ — за окупаемость и приоритет в выдаче.

## 2. Где мы сейчас

**Есть и работает:** кабинет разборки (авто-доноры со статусами разборки, склад с фото/категориями/иерархическими местами хранения, заказы с мультивалютностью UAH/USD, CRM клиентов, Trash); публичный маркетплейс /market (каталог всех разборок, фильтры, корзина, анонимная заявка по телефону → кабинет разборки); лендинг /business с заявкой на открытие; подписки в БД (Демо + 4 плана, флаг `has_analytics`, лимиты авто/запчасти/сотрудники с гейтингом `canCreate`); базовая аналитика; курс USD с ПриватБанка.

**Ключевые пробелы (фактура из кода):**
- `purchase_price` запчасти **срезается деструктурингом** в `createPartsInventoryItem` — окупаемость невозможна, данные не копятся.
- Лимит сотрудников на платных тарифах **сломан** (`useSubscriptionLimits` сверяет с `FREE_LIMITS` вместо `max_workers`); `max_customers` не применяется; в `PLAN_FEATURES` — мёртвые STO-тарифы.
- Заявка с маркета (`marketplace_orders`) — **тупик**: не конвертируется в `parts_order`, разборка пересобирает заказ вручную.
- Уведомлений нет вообще: ни о новых заявках, ни об истечении подписки (`isExpiringSoon` вычисляется, но никто не видит).
- Маркет не масштабируется: агрегации тянут до 5000 строк на клиент, `getMarketSuppliers` — N запросов; поиск — `ilike`; SEO нет; UUID в URL.
- Сотрудников нельзя пригласить/уволить из кабинета — раздел только просмотр.
- `telegram`/`description` разборки видны на маркете, но не редактируются в настройках; `min_stock_level` и `qr_code` не доведены до UI.
- Весь биллинг ручной (заявка → админ), онлайн-оплаты нет.
- Нет массового импорта склада — действующая разборка с 500+ позициями в Excel физически не мигрирует.

## 4. ТОП-10 инициатив (максимальный impact/effort)

1. **Волна 0: баг-долг** (`purchase_price`, лимит workers, PLAN_FEATURES, telegram/настройки, `created_by`) — S. Без этого ни окупаемость, ни тарифная лестница не существуют; данные для ROI должны копиться **с сегодняшнего дня**.
2. **Заявка маркета → заказ в 1 клик** — S. Замыкает уже построенный маркет на уже построенные заказы; делает ценность маркета измеримой («продали через /market на X ₴») — это удержание подписки.
3. **Telegram-бот уведомлений о заявках** — S–M. Скорость реакции = конверсия маркета; заодно решает «подписка истекает» (сейчас уведомлений нет вовсе). Рынок UA живёт в Telegram.
4. **Пейволлы + прогресс-бары + замки + гейт заявок для Демо** — S. Конверсия уже существующих демо-компаний в платящих; почти чистый фронтенд.
5. **Окупаемость авто (ROI)** — M (S после волны 0). Единственная причина платить 800₴; конкуренты этого не дают в принципе; все данные уже в БД.
6. **Импорт CSV/XLSX** — S–M. Главный барьер миграции действующих разборок; лимит на импорт = встроенный апсейл.
7. **Чек-лист онбординга + режим «Конвейер» + дублирование позиции** — S. Активация: путь «авто заехало → запчасти на маркете» с дней до часов.
8. **Фиды OLX/авто.ria + автопостинг в Telegram-канал** — M. Превращает продукт в «мастер-склад всех каналов»; естественная причина апгрейда 400→600₴.
9. **LiqPay self-serve** — M–L. Ручной биллинг — потолок ~20 компаний; пейволлы из п.4 без оплаты в один клик теряют импульс.
10. **Маркет-фундамент покупателя**: подбор по авто + tsvector-поиск + блок доверия на карточке + статус заявки по токену — M. Без этого маркет не даст заявок, на которых держатся крючки тарифов.

### Gated-крючки (за что реально платят на каждом шаге)

- **Демо → Старт (400₴):** заявки с маркета. Позиции видны всем, интерес считается («интересовались 14 раз»), но кнопка «В корзину» отвечает «разборка ещё не принимает заявки». Владелец видит упущенные деньги.
- **Старт → Бизнес (600₴):** объём (300 позиций ≈ 1,5 авто) + **фид OLX/авто.ria** (экономит часы ручного дублирования — понятная цена 200₴) + Telegram-бот вместо email + QR-этикетки.
- **Бизнес → Профи (800₴):** **окупаемость авто** («Camry 2012: куплена за $1 800, продано на $2 340, окупаемость 130%, на складе ещё $1 100») + приоритет в выдаче маркета — единственная ось, где тариф *приносит* деньги, а не экономит.
- **Профи → Премиум:** мультифилиал, API, закреп в топе, опт — под первого реального сетевого клиента.

### Механика апгрейда

1. **Self-serve оплата LiqPay** (у аудитории всегда есть Приват; продукт уже интегрирован с API ПриватБанка по курсу) + IBAN-инвойс для ФОПов; админ-аппрув остаётся только Премиуму.
2. **Пейволл в момент действия** с офером и кнопкой оплаты в 1 клик — ловим импульс.
3. **Прогресс-бары лимитов** (жёлтый 80% / красный 100%) — апгрейд до фрустрации.
4. **Замки с превью** на аналитике/окупаемости.
5. **Триал Профи 7 дней** после месяца на Старте.
6. **Годовой план −17%**; **add-ons** дороже за единицу, чем апгрейд (подталкивают к следующему тарифу).
7. **Якорение:** «Бизнес» помечен «Выбор большинства разборок»; Премиум всегда виден как якорь дороговизны.
8. **Даунгрейд без потери данных** (read-only) — обещано явно; грейс 3 дня после истечения, затем read-only, не удаление.

## 6. Фазовый план реализации

### Этап 1 — «Фундамент + ядро ценности» (~4–6 недель)
**Зачем:** технически включить тарифную лестницу, замкнуть цикл «маркет → деньги», начать копить данные для ROI, конвертировать существующие демо-компании.

- Волна 0 целиком (баг-долг: `purchase_price`, лимиты, PLAN_FEATURES, настройки telegram/description/город, `created_by`, RPC-агрегаты маркета, чистка legacy).
- Заявка маркета → заказ в 1 клик (+ кнопки Viber/Telegram на заявке).
- Telegram-бот: заявки с маркета + истечение подписки (pg_cron).
- Пейволлы-модалки, прогресс-бары, замки, гейт заявок для Демо + счётчик интереса; новая сетка лимитов (Демо = 50 запчастей / 14 дней).
- Онбординг: чек-лист первого входа, автокопирование категорий, демо-данные.
- Скорость ввода: режим «Конвейер», дублирование позиции, экспорт CSV.
- VIN-декодер.

**Монетизация этапа:** первые честные конверсии Демо→Старт (крючок «заявки»), удержание текущих платящих (бот, конвертация заявок).

### Этап 2 — «Продающая ценность тарифов + self-serve деньги» (~6–8 недель)
**Зачем:** дать причины платить 600₴ и 800₴ и убрать админа из цепочки оплаты.

- **Окупаемость авто** (страница авто + сводка маржи + `parts_vehicle_expenses`) + мёртвый сток — киллер-фичи Профи.
- **Импорт CSV/XLSX** — миграция действующих разборок (главный источник новых платящих).
- **Фиды OLX/авто.ria + автопостинг в Telegram-канал** — крючок Бизнеса.
- **LiqPay self-serve** + история платежей + cron-жизненный цикл (грейс → read-only).
- Приглашение/деактивация сотрудников + скрытие закупочных цен (RLS).
- Камера-first + миграция фото на Supabase Storage.
- Маркет-фундамент: подбор по авто, tsvector-поиск, блок доверия на карточке, статус заявки `/market/order/:token`, слаги в URL.

**Монетизация этапа:** апгрейды Старт→Бизнес (фиды) и Бизнес→Профи (ROI); самостоятельные оплаты без админа; приток мигрирующих разборок через импорт.

### Этап 3 — «Рост маркета и сетевые эффекты» (далее, итерациями)
**Зачем:** трафик покупателей и монетизация видимости — когда есть кому её продавать.

- SEO (пререндер, sitemap, OG) — после слагов из Этапа 2.
- «Разыскивается деталь», «Сообщить, когда появится», избранное.
- Новая Почта: API-ТТН из заказа.
- QR-этикетки + печать + скан-продажа; мультивыбор с массовыми действиями; поиск «где лежит» / страницы локаций.
- Отзывы по завершённым заявкам + автоматические бейджи → затем **приоритет в выдаче + поднятия + буст-add-on** (только при стабильном потоке заявок, иначе крючок пуст).
- Add-ons, годовые планы, триал Профи.
- Аккаунт покупателя + «Гараж», совместимость-лайт, автоподсказка цены, KPI сотрудников.

**Осознанно отложено:** чат покупатель↔разборка (до >50 активных разборок), полный TecDoc, эскроу/онлайн-оплата покупателя (наложка НП закрывает 90%), нативное приложение (PWA + сканер), мультифилиал и публичный API (под первого клиента Премиума), OLX Partner API (после валидации спроса фидами).

## 7. Метрики успеха

| Группа | Метрика | Цель / норматив |
|---|---|---|
| **Активация** | Демо-компания дошла до «1 авто + 10 запчастей с фото + позиции видны на маркете» за 14 дней | > 50% одобренных заявок |
| | Время «первое авто → первая запчасть с фото» | < 1 дня |
| **Конверсия в платный** | Демо → Старт за 30 дней после регистрации | > 25% |
| | Доля апгрейдов, прошедших self-serve (без админа) | > 80% после LiqPay |
| **Апгрейд-рейт / ARPU** | Старт → Бизнес+ в течение 90 дней | > 20% |
| | ARPU платящей разборки (база 400₴) | рост к 550–600₴ за счёт лестницы и add-ons |
| **Удержание** | Churn платящих в месяц | < 5%; продление после первого месяца > 85% |
| | Реактивация по грейс-уведомлениям бота | измерять с Этапа 2 |
| **GMV маркета** | Сумма заявок, конвертированных в заказы (`converted_order_id`) — становится измеримой после Этапа 1 | рост м/м; «продано через /market» в кабинете каждой разборки |
| | Доля заявок, доведённых до заказа/«обработана» | > 60% |
| | Median времени реакции new → viewed | < 2 ч (публикуем бейджем) |
| **Маркет-воронка** | Конверсия карточка → заявка; карточки с 3+ фото и OEM vs без | разрыв = аргумент для скоринга выдачи |
| | Органический трафик на карточки (после SEO) | главный опережающий индикатор Этапа 3 |
| | Повторная заявка с того же телефона/аккаунта за 90 дней | > 15% |

**Главный составной индикатор здоровья:** число разборок, у которых в месяц ≥ 1 заявка с маркета конвертирована в заказ И заполнена окупаемость хотя бы по одному авто — это пересечение двух ценностей, за которые платят, и лучший предиктор продления подписки.
