## Что уже есть (`get_parts_analytics`, `PartsAnalytics.tsx`)
Выручка с продаж, всего/завершённых заказов, продано запчастей, стоимость склада, средний чек,
разобрано авто, потенциальная маржа, выручка по месяцам (график), топ запчастей, топ авто-доноров.

## A. Добавить из УЖЕ имеющихся данных (сбор не нужен)
1. **Прибыль и маржа** — теперь есть `purchase_price` + формула прибыли: валовая прибыль (выручка − себестоимость), **маржа %**, прибыль по месяцам (вторая линия на графике выручки). По позициям с известной себестоимостью (как на дашборде).
2. **Переключатель периода** — Сегодня / 7д / Месяц / Квартал / Год / Всё (как на дашборде). Все метрики уважают период.
3. **Оборачиваемость склада** — средний срок «от добавления до продажи» (`created_at → sold_at`); сколько дней в среднем лежит до продажи.
4. **Неликвид / залежавшиеся** — запчасти в наличии дольше N дней (90/180) без продажи, с суммой замороженных денег. Топ-список «что распродать».
5. **Конверсия заказов** — доля `completed` от всех; доля `cancelled`; воронка new→assembling→shipped→completed.
6. **Топ категорий** — по выручке и по прибыли (а не только топ отдельных запчастей).
7. **Окупаемость авто сводно** — средний % окупаемости, самые прибыльные / самые «висящие» авто, среднее время полной разборки.
8. **Клиентская аналитика** — новые vs повторные клиенты, топ-клиенты по сумме, средний чек на клиента, доля повторных продаж.
9. **Доставка** — доля заказов с ТТН НП vs самовывоз; типичная сумма наложенного платежа.
10. **Динамика склада** — стоимость склада по месяцам (растёт/тает запас).

## B. Данные, которых пока нет — начать собирать
1. **Просмотры запчасти** — в БД уже есть `increment_part_views` (RPC). Подключить вызов на публичной странице запчасти → метрика «самые просматриваемые», конверсия **просмотр → продажа**. *(дешёвый и ценный сигнал спроса)*
2. **Спрос из маркета** — `marketplace_orders` + поиск по каталогу: что ищут/заказывают, чего **нет в наличии** (упущенный спрос). Нужно логировать запросы каталога (поисковые термины) или хотя бы заявки.
3. **Канал продажи** — добавить на заказ поле `channel` (маркет / звонок / на месте / Telegram) → выручка по каналам.
4. **Причина отмены** — поле при отмене заказа → почему теряем заказы.
5. **Себестоимость** — поле `purchase_price` есть, но 0% заполнено. Без него прибыль = 0. Нужно приучить заполнять (или подсказка/обязательность для магазинных запчастей).
6. **Дата полной разборки авто** — когда машина перешла в «Разобран» → точное «время разборки» и окупаемость по факту.
