# SEO-план razborka.net — что видит Google и что доделать (2026-06-28)

Проверено на проде запросом сырого HTML как Googlebot (без JS) + анализ `worker/index.js`, `robots.txt`, `sitemap.xml`, `index.html`.

---

## Как сейчас видит краулер (факты)

**✅ Уже работает (edge-SEO в воркере — сильная база):**
- Per-route `title` / `description` / `canonical` / OpenGraph / Twitter Card инжектятся на edge (HTMLRewriter) — поиск видит уникальные мета без SSR.
- JSON-LD: `Organization`, `WebSite`+`SearchAction`, `BreadcrumbList`, `Product`+`Offer`, `AutoPartsStore`.
- hreflang `ru`/`uk` (`?lng=uk`) + `x-default`, `og:locale`, динамический `<html lang>`.
- `sitemap.xml` из реальных данных (товары + поставщики), edge-кеш.
- `www` → apex 301, честный HTTP 404 на несуществующие сущности, корректный `robots.txt` (Disallow приватного, noindex тонкого).

**❌ Дыры (по убыванию важности):**
1. **Тело страницы пустое в сыром HTML.** Проверка: `<body>` → **0 символов** видимого текста, нет `<h1>`, нет внутренних ссылок, нет текста товара. Весь контент рисует React после загрузки JS. Googlebot JS рендерит (2-я волна, медленнее и с риском), **а Yandex/Bing/соц-боты — почти нет**. Для авторазборок в UA/RU Yandex — значимый канал → контент обязан быть в HTML.
2. **Нет SEO-страниц по марке/модели/категории** (`Tesla`, `Tesla Model 3`, «двигатели Tesla» …). Это главный органический хвост — сейчас его нет вообще. Каталог фильтруется только через query-параметры, не индексируется как отдельные страницы.
3. **Product JSON-LD неполный:** нет `offers.seller` (разборка-продавец), `aggregateRating`/`review` (**отзывы в БД уже есть** — `get_company_rating`!), `priceValidUntil`, `shippingDetails`, `hasMerchantReturnPolicy`. → нет шансов на rich-результаты/Merchant.
4. **sitemap:** без `lastmod`, без `uk`-альтернатив (`xhtml:link`), без картинок; один файл (лимит 50k URL — при росте нужен sitemap-index).
5. **Мелочи:** нет `image` sitemap для фото товаров; внутренняя перелинковка слабая (PageRank не растекается); нет текстовых описаний-«простыней» на категориях.

## План реализации

### P0 — Контент в HTML (рендеринг для краулеров) — **главное**
**Цель:** в сыром HTML появляются H1, ключевой текст и внутренние ссылки на всех публичных страницах.

1. **Серверный SEO-body в воркере.** Расширить `injectMeta`: помимо `<head>`, инжектить блок контента в `<div id="root">…</div>` (React при гидратации его заменит — пользователи не заметят, краулер без JS видит контент). Воркер уже тянет данные товара/поставщика через RPC — переиспользуем:
   - **Товар:** `<h1>{name}</h1>`, авто/состояние/цена/номер, описание, ссылка на поставщика, хлебные крошки-ссылки.
   - **Каталог/Главная:** `<h1>`, вводный абзац, ссылки на топ-марки/категории (перелинковка).
   - **Поставщик:** `<h1>{name}</h1>`, город/контакты, ссылка в каталог по этому поставщику.
   - Риск: `createRoot().render` очищает контейнер — ок (не `hydrateRoot`). Проверить, что нет FOUC/мерцания.
   - *Альтернатива при росте:* полноценный SSR/prerender (prerender.io или React SSR в воркере) — дороже, пока не нужно.

### P1 — Расширение охвата (органический хвост)
2. **SEO-страницы по марке/модели/категории.** Путь-ориентированные URL вместо query:
   - `/market/catalog/tesla`, `/market/catalog/tesla/model-3`, `/market/catalog/dvigateli` и т.п.
   - Клиент: роут рендерит отфильтрованный каталог; воркер: per-route мета + SEO-body (H1 «Запчасти Tesla Model 3», текст, `ItemList` JSON-LD, ссылки на под-категории).
   - Sitemap: добавить эти URL из справочника `car_models` + категорий.
   - Внутренние ссылки: на главной/каталоге — блок «Популярные марки», на товаре — «другие запчасти {make} {model}».
   - Это **L по объёму**, но главный источник трафика.
3. **Полный Product JSON-LD:**
   - `offers.seller` → `{ '@type': 'AutoPartsStore', name: <разборка> }` (имя уже есть в RPC `get_public_parts_item` — добавить компанию в выдачу RPC, если нет).
   - `aggregateRating` из `get_company_rating` (рейтинг разборки) — данные уже есть.
   - `priceValidUntil` (например +30 дней от now в воркере — передать timestamp в args, т.к. `Date` в воркере есть), `itemCondition` (есть), `availability` (есть).
   - Опц.: `shippingDetails` (Нова Пошта), `hasMerchantReturnPolicy` (гарантия разборки — поля `warranty_*` уже есть).
4. **Sitemap-апгрейд:**
   - `lastmod` (из `updated_at` товара/поставщика — добавить в select).
   - `uk`-альтернативы через `<xhtml:link rel="alternate" hreflang="uk" href="…?lng=uk">` в каждом `<url>`.
   - sitemap-index при приближении к 50k URL (разбить на `sitemap-parts-N.xml`).
   - image-sitemap: `<image:image>` с фото товара.

### P2 — Полировка
5. **Перелинковка и текст:** SEO-тексты (200–400 слов) на категориях/марках; блок «Похожие запчасти» на товаре (внутренние ссылки).
6. **Core Web Vitals:** добить lazy/размеры на `<img>` (CLS), проверить LCP карточек каталога; INP SPA.
7. **`og:image` на товар без фото** — генерить динамичную OG-картинку (имя+цена) или оставить дефолт (сейчас дефолт — приемлемо).
8. **Yandex-специфика:** `Clean-param` в robots для трекинг-параметров; Яндекс.Вебмастер + IndexNow (Bing/Yandex быстрый пинг при появлении товара).
9. **Аналитика индексации:** Google Search Console + Bing Webmaster + Яндекс.Вебмастер — submit sitemap, отслеживать покрытие/ошибки.

## Порядок работ (рекомендация)
1. **P0.1** серверный SEO-body (товар → каталог/главная → поставщик) — даёт контент в HTML сразу всем краулерам.
2. **P1.3** полный Product JSON-LD (быстро, данные есть) + **P1.4** sitemap lastmod/hreflang.
3. **P1.2** SEO-страницы марка/модель/категория — крупнейший канал, делать отдельным этапом.
4. **P2** полировка + вебмастер-консоли + IndexNow.

> Зависимости/ноды: всё ложится в существующий `worker/index.js` (RPC `get_public_parts_item`, `get_market_suppliers`, `get_company_rating`, справочник `car_models`). Новых внешних сервисов не требуется (кроме опц. вебмастер-консолей).

## РЕАЛИЗОВАНО (2026-06-28)

- **P0 — серверный SEO-body** в воркере (`injectMeta` → инъекция в `#root`): главная, каталог, лендинги марка/модель, товар (H1 + фото + характеристики + описание + ссылка на продавца), поставщик, авторазборки, для-бизнеса. React (`createRoot`) очищает контейнер при гидратации — пользователь видит обычное приложение, краулер без JS видит контент.
- **P1.3 — полный Product JSON-LD**: `offers.seller` (AutoPartsStore), `aggregateRating` (из `get_company_rating`), `priceValidUntil` (+30 дн.), `shippingDetails` (UA, 1–3 дня), `warranty` (по `warranty_*` разборки). RPC `get_public_parts_item` расширен полями `company` + `rating` (миграция `2026-06-28_public_item_seo_fields.sql`).
- **P1.4 — sitemap**: `lastmod` (created_at), hreflang-альтернативы `ru`/`uk`/`x-default` (`xhtml:link`), image-sitemap (`image:image`), + URL лендингов марок/моделей.
- **P1.2 — SEO-страницы марка/модель**: пути `/market/catalog/:make[/:model]` — воркер (мета + body + breadcrumb + 404 на пустые), клиент (`MarketCatalog` резолвит slug→фильтр, H1/мета), sitemap, перелинковка (чипы марок на главной → лендинги). Slug-утилита `src/utils/slug.ts` ↔ `slugify` в воркере.
- **P2 — Yandex Clean-param** (схлопывание utm/fbclid/gclid/yclid; `?lng` не трогаем). **IndexNow** — эндпоинт `/api/indexnow` готов (защищён `CRON_RATE_SECRET`).

### Осталось активировать вручную (как LiqPay-ключи)
- **IndexNow** (мгновенная индексация Bing/Yandex): (1) сгенерировать ключ, положить `public/<key>.txt` со значением ключа; (2) задать секрет воркера `INDEXNOW_KEY`; (3) Supabase Database Webhook на `parts_inventory` INSERT (status=available) → `POST https://razborka.net/api/indexnow` с телом `{ "secret": "<CRON_RATE_SECRET>", "urls": ["/market/part/<id>"] }`.
- **Вебмастер-консоли**: Google Search Console + Bing Webmaster + Яндекс.Вебмастер — подтвердить домен, отправить `sitemap.xml`.

### Возможные дальнейшие шаги (P2+, не критично)
- SEO-страницы по категориям (`/market/catalog/cat/<slug>`) — аналогично марке/модели.
- SEO-тексты-«простыни» на лендингах (200–400 слов), блок «Похожие запчасти» на товаре.
- sitemap-index при приближении к 50k URL.
