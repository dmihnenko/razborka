# SEO-аудит razborka.net — 2026-06-26

Проверено: код (index.html, public/, vite.config, зависимости, per-route мета, JSON-LD) + прод (сырой HTML краулера, robots.txt, sitemap.xml, заголовки).

## Главный вывод
razborka.net — **client-rendered React SPA без SSR/пререндера**. Краулер на ЛЮБОЙ URL получает один и тот же пустой `index.html`-шелл с **одинаковым** generic-заголовком; контент (товары, разборки) появляется только после выполнения JS. Для маркетплейса, который должен находиться в поиске по конкретным запчастям/городам, это критично.

---

## Находки (по severity)

### 🔴 SEO-1. Нет per-route мета (title/description/OG/canonical)
Сырой HTML `/`, `/market/catalog`, `/market/product/...` — **один и тот же** `<title>Razborka.net — маркет б/у запчастей…</title>` и одно `description`. Нет `react-helmet` (в зависимостях отсутствует), мета не меняется по маршруту.
- **Эффект:** Google не покажет уникальный сниппет на товар/разборку; ранжирование по конкретным запчастям слабое; **шеринг ссылок** (Telegram/WhatsApp/FB — а это основной канал маркета, у вас есть кнопка «Поделиться») показывает **одинаковое generic-превью** без фото/названия товара.
- **Фикс:** уникальные `title`/`description`/`canonical`/Open Graph на каждую публичную страницу. Два пути (лучше оба): (а) `react-helmet-async` — клиентские теги (Googlebot их рендерит, но соц-краулеры — нет); (б) **инъекция мета на edge в Cloudflare Worker** (`worker/index.js`) по `pathname` — тогда и соц-краулеры видят корректные OG. Для товара тянуть title/desc/og:image из публичного RPC.

### 🔴 SEO-2. Контент не виден краулеру (нет пререндера/SSR)
Сырой ответ — пустой шелл; список товаров/карточка/разборка рисуются клиентом. Googlebot JS рендерит (с задержкой и не всегда), Bing/соц-сети/прочие — в основном нет.
- **Фикс:** пререндерить **публичные** страницы (маркет-home, каталог, карточка товара, разборка, публичные QR-страницы запчасти/места/клиента). Варианты: `vite-ssg`/`vite-plugin-prerender` на сборке для статичных маршрутов; для динамических (товары) — **edge-SSR/пререндер в Worker** или периодическая генерация. Кабинет/админка SEO не нужны (за авторизацией) — фокус только на публичном периметре.

### 🟠 SEO-3. Нет настоящего sitemap.xml
`/sitemap.xml` на проде отдаёт **SPA-заглушку** (`<!doctype html>`, т.е. SPA-фолбэк, файла нет). `robots.txt` без директивы `Sitemap:`.
- **Эффект:** краулер не обнаруживает каталог товаров/разборок (их тысячи, на них нет ссылок из статичного HTML).
- **Фикс:** генерировать **реальный** `sitemap.xml`: статичные маршруты + динамические URL товаров/разборок из БД (на сборке или Worker-роутом, периодически). Добавить `Sitemap: https://razborka.net/sitemap.xml` в `robots.txt`. Убедиться, что `/sitemap.xml` — реальный ассет (не перехватывается SPA-фолбэком).

### 🟠 SEO-4. Нет структурированных данных (JSON-LD)
Нет `Product`/`Offer`, `Organization`, `BreadcrumbList`, `LocalBusiness` — ноль шансов на rich results.
- **Фикс:** JSON-LD: `Organization` (home), `Product`+`Offer` (карточка товара: название, цена, валюта, состояние, наличие, фото), `BreadcrumbList` (каталог/товар), `LocalBusiness` (страница разборки: имя, город, телефон). Это даёт расширенные сниппеты (цена/наличие/рейтинг) в выдаче.

### 🟡 SEO-5. robots.txt минимальный
Сейчас только `User-agent: * / Allow: /`. Нет `Sitemap:`, нет `Disallow` приватных/неиндексируемых путей.
- **Фикс:** добавить `Sitemap:`; `Disallow` для `/parts/`, `/admin`, `/welcome`, `/reset-password`, `/vehicle-access`, кабинетных/аккаунтных путей — чтобы не тратить crawl-budget и не индексировать тонкие auth-страницы.

### 🟡 SEO-6. Нет canonical
Риск дублей: `?filter=` параметры каталога, www/non-www, trailing slash.
- **Фикс:** self-referencing `<link rel="canonical">` на каждую публичную страницу; редирект www→apex (или наоборот) на Cloudflare.

### 🟡 SEO-7. i18n без hreflang / locale в URL
Двуязычность ru/uk хранится в БД/cookie, **не в URL** → для поиска индексируется фактически одна версия, нет `hreflang`-альтернатив.
- **Фикс (по желанию):** либо вынести локаль в URL (`/uk/...`) + `hreflang`, либо осознанно оставить RU как индексируемый и не плодить дубли. Минимум — корректный `<html lang>` (сейчас всегда `ru`, даже для uk-посетителя).

### 🟢 Что хорошо
- `robots.txt` есть (Allow all), `<html lang="ru">`, `viewport`, `description` присутствуют; чистые URL (`/market/catalog`, `/market/product/...`); HTTPS + security-заголовки; `theme-color`, PWA-манифест, mobile-friendly. Каркас для SEO есть — не хватает per-route мета, пререндера, sitemap, JSON-LD.

## Приоритет внедрения (по ROI для маркета)

**P0 — максимальная отдача, относительно дёшево:**
1. **Edge-инъекция мета в `worker/index.js`**: по `pathname` подставлять `<title>`/`description`/`canonical`/OG (`og:title`/`og:description`/`og:image`/`og:url`/`og:type`) для публичных маршрутов (home, catalog, product, supplier, public part/location). Для товара/разборки — короткий публичный запрос за полями. Это разом чинит SEO-1 (мета) и шеринг-превью для соц-краулеров, не требуя полного SSR.
2. **Реальный `sitemap.xml`** (статика + динамика из публичного RPC) + `Sitemap:` в `robots.txt`.
3. **JSON-LD** `Product`/`Organization`/`LocalBusiness` (через ту же edge-инъекцию или react-helmet).

**P1:**
4. Пререндер публичных страниц (vite-ssg для статичных; edge-SSR контента товара — если P0-мета окажется недостаточно для ранжирования контента).
5. `robots.txt`: `Disallow` приватных путей; canonical; редирект www.

**P2:**
6. Стратегия i18n-URL + hreflang. Перф/CWV (вынести крупный vendor-чанк ~1.45MB — влияет на LCP).

## Razborka.net-специфика
- Деплой — **Cloudflare Worker + Static Assets** (`worker/index.js` уже есть и проксирует `/api/*`). Это **идеальное место для edge-мета/OG-инъекции и Worker-роутов `/sitemap.xml`** без полного SSR-фреймворка — самый дешёвый путь к P0.
- Публичные данные уже доступны анонимно через `SECURITY DEFINER` RPC (`get_public_parts_item`, `get_market_*`) — Worker может дёргать их для мета/sitemap без раскрытия приватного.
- Кабинет/админка/«Мои авто» — за авторизацией, в индекс не нужны (только `Disallow`).
