# Промпт: оформление маркета автозапчастей (razborka.net / market)

> Готовый бриф для дизайнера или ИИ-агента. Цель — отполированный, **светлый**, красивый и
> конверсионный публичный маркетплейс Б/У и новых автозапчастей от разборок. Русский UI,
> mobile-first. Без необрутализма и тёмных тем. Все цвета/тени/радиусы — из токенов `:root`
> (`src/index.css`), без хардкода hex/px в компонентах.

---

## 1. Контекст и цель
- **Продукт:** публичный маркетплейс запчастей. Покупатель ищет конкретную деталь под своё авто
  (часто по OEM-номеру или Год/Марка/Модель), смотрит фото, цену, состояние, доверяет продавцу-разборке.
- **Поведение:** покупатели **ищут, а не листают** → поиск по авто (YMM) и по OEM — первичны.
- **Аудитория:** автовладельцы и СТО; гости (без логина) и залогиненные.
- **Метрика успеха:** найти нужную деталь за минимум кликов; доверие (фото, рейтинг, гарантия,
  доставка Новой Почтой) → добавление в корзину/заявка.

## 2. Арт-направление (выбрать одно как базу, остальное — донор идей)
Светлая база обязательна. Кандидаты (исследованы отдельно): **Минимал-люкс**, **Маркетплейс-ретейл
(Rozetka/OZON)**, **Тёплый бутик**, **Тех-дашборд (Stripe/Linear)**, **Журнальный (editorial)**.
По умолчанию — чистый светлый каталог с чёткими 1px-границами и аккуратными тенями. Характер задаётся
композицией и типографикой, цвет — точечно.

## 3. Дизайн-система (точные значения)
**Палитра (текущая, светлая):**
- Фон холста `#F6F7F9`, поверхность-карточка `#FFFFFF`, лёгкая заливка/hover `#EEF0F3`.
- Текст `#111827` / вторичный `#4B5563` / мета `#6B7280`.
- Границы `#E5E7EB`, усиленная `#D5D9DF`.
- **Акцент-сталь `#4F5B7A`** (hover `#3E4860`, weak `rgba(79,91,122,.10)`) — основной акцент/иконки.
- **Бренд-синий `#1173D4`** — только на главных действиях («Купить», «Подобрать»).
- Семантика: «в наличии/Новое» зелёный `#16A34A`; «Б/У» нейтральный/янтарный `#B45309`; скидка/ошибка `#DC2626`.
- Допустим **один** тактичный доп-акцент под арт-направление (напр. тил/терракота) — точечно.

**Типографика:** UI-гротеск (Inter/Manrope) + **JetBrains Mono для OEM/VIN/цен** (tabular-nums).
Заголовки плотные, `letter-spacing` слегка отрицательный. Цена — якорь иерархии.

**Радиусы:** кнопки/инпуты/чипы 7px, карточки/панели 12px, секции/модалки 16px, пилюли 999px.
Скругления концентричные (вложенный = внешний − паддинг).

**Тени (мягкие, многослойные, холодные):**
- покой `0 1px 2px rgba(17,24,39,.04), 0 1px 3px rgba(17,24,39,.05)`
- подъём/hover `0 6px 16px -8px rgba(17,24,39,.12)`
- поповер/модалка — глубже, но деликатно. Чёрных/жёстких offset-теней нет.

**Границы:** у карточек/панелей виден **1px бордер** (не только тень). Усиление на hover — **цветом**
бордера, не толщиной. Внутренние разделители — светлее внешних.

**Сетка/ширина:** контейнер до `1920px` (`--mk-maxw`), fluid-паддинги; десктоп каталог 3–5 колонок,
планшет 3, мобайл 2. Тач-таргеты ≥ 44px. Контраст ≥ WCAG AA.

**Состояния:** всегда продумать loading (скелетоны), пусто (empty-state с иконкой и подсказкой),
ошибку, «нет фото» (иконка + подпись), disabled. Фокус — видимое кольцо `0 0 0 3px var(--mk-focus)`.

## 4. Меню и навигация
**Десктоп-шапка (sticky, белая, 1px-низ):**
- Слева лого «Маркет запчастей».
- Центр: строка поиска (по названию/OEM) + табы **Каталог**, **Разборки**.
- Справа: иконки **Поиск** (моб.), **Корзина** (с бейджем количества), ссылка **«Открыть разборку»** (для бизнеса).
- Под шапкой на главной/каталоге — **поиск по авто: Год · Марка · Модель + «Подобрать»** и/или
  ряд категорий-плиток (Двигатель, КПП, Оптика, Кузов, Подвеска, Салон, Электрика, Тормоза).

**Мобильная навигация:** компактная шапка (лого + поиск-иконка + корзина); табы Каталог/Разборки
строкой; поиск по авто — в bottom-sheet. Не дублировать поиск в шапке, если он есть на странице.

**Футер:** короткий — лого, «Маркет запчастей · Б/У и новые от разборок», ссылка «Открыть разборку»,
контакты/город. Без тёмной заливки.

## 5. Компоненты
**Карточка товара (`MarketProductCard`) — ключевой элемент:**
- Фото 4:3 на светлой подложке (`object-contain`, чтобы Б/У-снимки с разным фоном не ломали сетку);
  бейдж состояния (Б/У/Новое) в углу фото.
- Название в 2 строки (line-clamp); **OEM** моноширью (компактный чип); авто-донор (марка/модель/год)
  с иконкой; продавец/город — мелкой метой.
- Подвал: **цена** (₴, при USD — эквивалент) + **одна** явная кнопка действия («+»/«В корзину»).
- Hover: подъём тени + усиление бордера цветом. Кликабельна целиком → страница товара.
- Без конкурирующих действий (не пихать «сравнить/избранное/быстрый просмотр» рядом с «В корзину»).

**Бейджи:** `Б/У` (нейтр), `Новое` (зелёный), `Оригинал`, `Гарантия 14 дней`, скидка `−15%` (красный).
**Кнопки:** primary-сталь, brand-синий (Купить), outline, ghost, icon-add (круглый сталь).
**Фильтры (`FilterBar`):** марка/модель/год/состояние/цена/город; чипы + раскрываемые секции;
на мобайле — bottom-sheet, на десктопе — левый сайдбар или верхний бар.
**Карточка продавца (`SellerContactCard`/`SupplierCard`):** название разборки, город, **рейтинг
звёздами + кол-во сделок**, телефон/Telegram, кнопка «Связаться».
**Айтем корзины:** фото-мини, название, продавец, цена × кол-во, степпер, удалить.

## 6. Страницы (детально)

### 6.1 Главная (`/market` → `MarketHome`)
- Герой: короткий заголовок + **поиск по авто (YMM)** и/или строка поиска по OEM, без воды.
- Ряд **категорий-плиток** (иконки lucide).
- Секции: «Новые поступления», «Подборки по марке», возможно «Топ разборок».
- Карточки — компонент `MarketProductCard`. Скелетоны при загрузке, empty-state если пусто.
- Без длинного маркетингового текста: сразу к поиску и товарам.

### 6.2 Каталог / листинг (`/market/catalog` → `MarketCatalog`)
- Заголовок «Каталог запчастей» (без подзаголовка-воды).
- **Фильтры** (`FilterBar`): марка/модель/год/состояние/цена/город + сортировка + переключатель
  сетка/список. Активные фильтры — чипами со сбросом.
- Сетка карточек 2 (моб) → 3–5 (десктоп), ленивая подгрузка/пагинация.
- Состояния: загрузка (скелетон-сетка), «ничего не найдено» (сброс фильтров), пусто.
- Хлебные крошки (Каталог / Категория / …).

### 6.3 Страница товара (`/market/part/:id` → `MarketProductPage`) — PDP
- **Галерея** над сгибом: крупное фото (zoom/lightbox) + ряд кликабельных миниатюр (не обрезать).
- Название (описательное), бейджи (Б/У · Оригинал · В наличии).
- **OEM-номер** моноширью + кнопка **«Копировать»**.
- **Крупная цена** (₴ + эквивалент $), пометка «курс фиксируется при оформлении».
- **CTA:** «В корзину» (сталь) + «Купить сейчас» (бренд-синий) — заметные, sticky на мобайле.
- **Авто-донор:** марка/модель/год/двигатель/пробег/**VIN** (моно).
- **Продавец-разборка:** карточка с рейтингом-звёздами, городом, телефоном, «Связаться».
- **Доставка/возврат:** Новая Почта, гарантия 14 дней на проверку.
- Блок описания/примечаний (если есть), похожие детали этой же разборки/донора.
- Состояния: «запчасть не найдена/продана», нет фото.

### 6.4 Разборки (`/market/suppliers` → `MarketSuppliers`)
- Сетка карточек разборок (`SupplierCard`): название, город, рейтинг, кол-во позиций/сделок, аватар/лого.
- Поиск/фильтр по городу. Empty-state.

### 6.5 Страница разборки (`/market/supplier/:id` → `MarketSupplierPage`)
- Шапка профиля разборки: название, город, рейтинг, контакты, краткое описание.
- Под ней — её товары (та же сетка карточек) с мини-фильтром.

### 6.6 Корзина (`/market/cart` → `MarketCart`)
- Список айтемов (сгруппировать по разборке-продавцу), степперы, удаление, подытоги по продавцу.
- Сводка: сумма, переход к **оформлению заявки** (RPC `submit_marketplace_order`): имя, телефон,
  комментарий → заявка падает разборке (в «Заявки с маркета»).
- Пустая корзина — дружелюбный empty-state с кнопкой «В каталог».

### 6.7 Сквозные
- Скелетоны для всех списков; тосты успеха/ошибки; sticky-CTA на мобайле; доступность (ARIA, фокус,
  клавиатура); адаптив 360px → десктоп.

## 7. Данные/контент (реальные, не пустые)
Использовать реальные/демо-данные (пример — Tesla от «Dnipro Tesla»): осмысленные названия деталей,
**настоящие OEM**, цены ₴/$, авто-донор с VIN, рейтинг продавца, **реальные фото** (не плейсхолдеры).
Никаких «Lorem»/пустых карточек в макетах для показа.

## 8. Тех-ограничения
- Цвета/тени/радиусы/шрифты — только через токены `--mk-*` / `--brand-*` из `src/index.css`.
- Готовые классы: `mk-card`, `mk-badge*`, `mk-price`, `mk-btn*`, `mk-add`, `mk-chip`, `mk-input`,
  `mk-tile`, `mk-container`. Не плодить новые цвета.
- Русский UI; деньги — `₴`, `toLocaleString('ru-RU')`; иконки `lucide-react` (stroke 1.5).
- PWA, mobile-first, перф (ленивые фото `srcSet`, скелетоны, без лишних перерисовок).

## 9. Что вернуть
Для каждой из 6 страниц + ключевых компонентов: отполированный экран в выбранном арт-направлении,
светлый, с чёткими границами и мягкими тенями, на реальных данных/фото, со всеми состояниями
(loading/empty/error). Показать десктоп и мобайл. Никакого необрутализма и тёмных плоскостей.
