# Спецификация-промпт: улучшения формы добавления запчасти

> Черновик на согласование. **Правь прямо в файле** — отметь, что брать (✅/❌).
> Источник: `PartsInventoryModal` в [src/pages/PartsInventory.tsx](../src/pages/PartsInventory.tsx)
> (≈1740–2464). Фоновая загрузка фото уже укреплена отдельно
> ([photo-upload-background-spec.md](./photo-upload-background-spec.md)).

## Что есть сейчас (факты)
Модалка на весь экран на мобиле / `max-w-5xl` на десктопе. Переключатель
**Одна запчасть / Список**. Авто-напоминание о последнем авто+месте (≈1878). Поля
одиночной формы: авто-донор, категория (каскад от авто), название\*, OEM (+копирование),
состояние\*, описание, кол-во (скрыто если выбрано авто), цена продажи + валюта,
закупочная цена (опц.), кнопка «Продано», место хранения, фото (до 5, фон). Массовый
режим: общие поля + таблица (название/цена/OEM/статус) + вставка из буфера.

---

## ✅ Реализовано (заход 1)
Блок A целиком + два запроса пользователя:
- A1 автофокус на «Название», A2 «Сохранить и добавить ещё» (липкие авто/категория/
  состояние/валюта/место; защита от попадания фото прошлой позиции в новую),
  A3 `inputMode` на числах, A4 валидация имени по trim, A5 липкие валюта+состояние
  (sessionStorage), A6 предупреждение об отрицательной марже.
- **Баг «0 нельзя удалить»** в числовых полях (кол-во/цена) — пустое значение больше не
  схлопывается в 0.
- **Скрытие закупочной цены для запчасти с авто** — она БУ с разборки, себестоимость от
  авто; при привязке к авто поле прячется и значение очищается.

---

## A. Быстрые победы (дёшево, заметный эффект)

1. **Автофокус на «Название»** при открытии (single) — сразу печатать, не тапать.
   Сейчас фокуса нет.
2. **«Сохранить и добавить ещё»** в одиночном режиме — вторая кнопка, которая
   сохраняет и **очищает форму, НЕ закрывая модалку** (авто/категория/место/валюта/
   состояние — сохранить как «липкие», обнулить только название/OEM/цену/фото). Для
   разборки, где вбивают десятки позиций подряд, это главный ускоритель. Сейчас каждое
   сохранение закрывает модалку (≈302–306).
3. **`inputMode="decimal"`** на ценах (продажа ≈2301, закупка ≈2325, кол-во) — на
   мобиле сразу числовая клавиатура. (OEM-поля уже имеют `autoCapitalize`.)
4. **Валидация названия по trim** — сейчас `required` пропускает строку из пробелов
   (single ≈2218). В `handleSubmit` проверить `formData.name.trim()` и показать ошибку.
5. **«Липкая» валюта и состояние** — запоминать последний выбор (как уже сделано для
   авто/места, ≈1746). На разборке валюта обычно одна на всю сессию.
6. **Предупреждение об отрицательной марже** — если `selling_price < purchase_price`,
   показать инлайн-подсказку под ценой («ниже закупки»). Дёшево, ловит опечатки.

## B. Ускорение заполнения (среднее, высокая ценность)

7. **Автоподстановка по OEM.** После ввода OEM искать в своём инвентаре позицию с тем
   же `part_number`: если нашлась — предложить подставить **название и категорию**
   («Уже было: Капот · Кузов — подставить?»). Убирает повторный ручной ввод и
   расхождения в названиях. (Запрос дешёвый — по индексу part_number в рамках компании.)
8. **Автокомплит названия** — `<datalist>` из ранее введённых названий запчастей
   компании (частые: «Капот», «Дверь передняя левая»…). Единообразие + скорость.
9. **Предупреждение о дубле OEM** — если такой OEM уже есть в наличии, мягко подсветить
   («такой артикул уже на складе») — не блок, а сигнал. Связано с п.7.
10. **Закупочную цену — приучить заполнять.** Сейчас заполнено 0% → прибыль/маржа в
    аналитике = 0 (см. [analytics-spec](./analytics-spec.md) B5). Не делать
    обязательной, но: подсказка «нужна для прибыли», и «липкое» напоминание, если
    магазинная запчасть без закупки. (Жёсткость — на твоё усмотрение.)

## C. Фото (среднее)

11. **Кнопка «Сделать фото» с `capture="environment"`** рядом с «Загрузить» — прямой
    выезд камеры на телефоне (быстрее медиатеки). Сейчас один input `accept="image/*"`
    (≈2386), камера — через системный диалог.
12. **Drag-&-drop фото на десктопе** в зону загрузки (сейчас только клик).
13. **Первое фото = обложка, с возможностью переставить** (drag-reorder превью). Сейчас
    порядок = порядок загрузки, обложку не выбрать.

## D. Консистентность и a11y (низкое-среднее)

14. **Статусы при создании single.** Сейчас в одиночном режиме можно отметить только
    «Продано» (≈2336), а в массовом доступны все 4 (available/reserved/sold/damaged,
    ≈2135). Унифицировать — дать в single тоже выбор статуса (или хотя бы +«Бронь»).
15. **Связать label↔input** (`htmlFor`/`id`) — сейчас `form-label` это просто класс без
    привязки, экранный диктор не озвучивает поле по клику на подпись. Плюс
    `aria-required` на обязательных.
16. **Функциональные `setFormData(prev => …)`** вместо `{ ...formData }` (≈2174 и др.) —
    убирает риск гонок состояния при быстрых правках. Чисто технический долг, без видимых
    изменений.

## Приоритет (моя рекомендация)
**Сначала A (1–6)** — дёшево, сразу ускоряет ежедневный ввод, без новых запросов/БД.
**Потом B7–B8** (автоподстановка по OEM + автокомплит названия) — наибольший выигрыш по
скорости, но нужен запрос к инвентарю. **C11** (камера) — если много вводят с телефона.
Остальное — по желанию.
