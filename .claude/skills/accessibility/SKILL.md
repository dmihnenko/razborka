# Доступность / a11y (этот проект)

Дополняет [ui-ux-designer]: тот отвечает за консистентность стиля, этот — за то, чтобы интерфейсом можно было пользоваться с клавиатуры, скринридером и при слабом зрении. Стек — React+TS+Tailwind, mobile-first, русский UI.

## Базовые правила

### Семантика и интерактивность
- Кликабельный элемент — `<button>`/`<a>`, не `<div onClick>`. Если действие навигация — `<a>`/`<Link>`; если действие в пределах страницы — `<button type="button">`.
- У иконочных кнопок (только иконка, без текста) ОБЯЗАТЕЛЕН `aria-label` (как в `InventoryCard`: `aria-label="Удалить"`). Декоративные иконки рядом с текстом — `aria-hidden`.
- Заголовки по иерархии (h1→h2→h3), не прыгать ради размера — размер задаётся классом.
- Списки — `<ul>/<li>`, таблицы данных — `<table>` с `<th scope>`.

### Клавиатура и фокус
- Всё интерактивное достижимо Tab'ом и срабатывает Enter/Space. Не ставь `tabIndex` > 0.
- **Модалки/bottom-sheet:** фокус уходит внутрь при открытии, Esc закрывает, фокус возвращается на триггер при закрытии. Фон — `aria-hidden` или focus trap.
- Не убивай видимый фокус: если переопределяешь `outline`, дай заметную замену (ring). Наш `:focus` на инпутах уже даёт `box-shadow` ring — сохраняй такой паттерн.
- `tabIndex={-1}` уместен только для управляемых программно элементов (как кнопка показать/скрыть пароль, чтобы не ломать Tab-поток поля).

### Контраст и размер
- Текст: контраст ≥ 4.5:1 (крупный ≥ 3:1). Осторожно с серым на белом: `text-gray-400` на `bg-white` ≈ на грани — для важного текста бери `text-gray-500/600+`. Подписи-капсы `text-[10px] text-gray-400` ок как вторичные метки, но не для смысловой информации.
- Тач-таргеты ≥ 44px (в base уже есть `btn-touch`); на мелких иконочных кнопках проверяй реальную область клика.
- Тёмная тема (`.dark`): проверяй контраст в обоих режимах. Инлайн `style={{color}}` тему не получает — частый источник нечитаемого текста в dark.

### Формы
- Каждый input связан с `<label>` (`htmlFor`/`id`) — не только плейсхолдер (плейсхолдер исчезает и низкоконтрастен).
- Ошибки валидации: текстом рядом с полем + `aria-invalid`/`aria-describedby`, не только цветом/тостом.
- Группы (радио/чекбоксы) — `<fieldset>`/`<legend>` или `role="group"` с `aria-label`.

### Изображения и медиа
- Содержательные `<img>` — осмысленный `alt` (у нас уже `alt={item.name}`); декоративные — `alt=""`.
- Не передавай смысл только цветом (статусы) — дублируй текстом/иконкой (бейджи уже с текстом — хорошо).

## Метод проверки экрана
1. Пройди Tab'ом сверху вниз: всё достижимо, порядок логичен, фокус виден?
2. Иконочные кнопки: у всех есть `aria-label`?
3. Модалка: Esc, focus trap, возврат фокуса?
4. Контраст ключевого текста в light и dark?
5. Формы: label + доступная ошибка?

## Чек перед сдачей
- Нет `<div onClick>` для действий.
- Все иконки-кнопки подписаны.
- Видимый фокус сохранён.
