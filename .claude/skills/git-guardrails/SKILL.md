# Git-страховка (этот проект)

Скил-СЕТАП: по запросу ставит `PreToolUse`-хук, который блокирует опасные git-операции до их выполнения. Сам по себе ничего не блокирует, пока не настроен.

## Что блокируем
`git push`, `git reset --hard`, `git clean`, `git branch -D`, `git checkout --`/`git restore` (отбрасывают изменения) и подобные необратимые.

## Согласовано с правилами репо
В проекте уже действует правило **«push — только по явной команде»** (см. `CLAUDE.md` и память [[push-only-on-request]]). Хук — техническая страховка этого правила. ВАЖНО: настрой паттерны так, чтобы при явном разрешении пользователя push оставался возможным (временно снять/обойти по согласованию), иначе хук заблокирует и санкционированный пуш.

## Установка (по запросу пользователя)
1. Уточни охват: проектный (`.claude/settings.json`) или системный (`~/.claude/settings.json`).
2. Положи bash-скрипт `block-dangerous-git.sh` в соответствующий `.claude/hooks/`.
3. Пропиши `PreToolUse`-хук типа `command` с матчером на Bash/PowerShell, вызывающий скрипт.
4. Скрипт разбирает команду; при совпадении с чёрным списком — `exit 2` и сообщение, что у агента нет прав на эту операцию.
5. Предложи кастомизацию списка паттернов.
6. Проверь тестовой командой, что блокировка срабатывает и обычные git-операции (status/add/commit) проходят.
