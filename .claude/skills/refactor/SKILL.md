# Рефакторинг (этот проект)

Цель — чище и консистентнее **без изменения наблюдаемого поведения**. Если поведение должно поменяться — это не рефакторинг, вынеси отдельно и подтверди.

## Принципы
1. **Сначала пойми.** Прочитай код и его использования (Grep по символу). Не трогай то, что не понял.
2. **Переиспользуй существующее, а не плоди новое:**
   - UI — дизайн-токены из `src/index.css` (`btn-*`, `card`, `modal-*`, `badge-*`) вместо повторяющегося сырого Tailwind.
   - Данные — общие функции в `src/services/*`, хуки в `src/hooks/*` вместо копипасты запросов.
   - Типы — `src/types/*` вместо локальных дублей.
3. **Малые шаги.** Изменение за изменением, каждое оставляет сборку зелёной. Не смешивай рефакторинг с фичами в одном заходе.
4. **Сохрани контракты:** сигнатуры экспортов, формы данных, queryKey react-query, роуты. Если меняешь queryKey — обнови все места и инвалидации.
5. **Чистка по ходу:** убирай реально мёртвый код и неиспользуемые импорты в файлах, которые трогаешь (проверь Grep'ом, что символ нигде не используется). Не вводи новых unused-locals.

## Чего избегать
- Массовых «причёсываний» всего репозитория за раз — только нужный охват.
- Изменения публичного поведения, текстов UI, порядка, дефолтов «заодно» — это требует согласования.
- Переписывания на новые паттерны/библиотеки без причины и согласования.

## Проверка
- `npx vite build` проходит. Если есть тесты на затронутый код — `npm test` зелёный.
- Перечисли, что изменилось и почему это безопасно (поведение прежнее).
