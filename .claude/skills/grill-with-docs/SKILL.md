---
name: grill-with-docs
description: Грилл-сессия, которая проверяет план против существующей доменной модели проекта, заостряет терминологию и обновляет доки по ходу. Используй, когда нужно стресс-тестить план против языка и зафиксированных решений проекта.
---

Проводи структурированное интервью по плану, сверяя его с уже зафиксированным в проекте знанием.

- **По одному вопросу за раз**, дожидаясь ответа, спускаясь по дереву решений.
- **Доки прежде вопросов.** Где можно — сверься с источниками правды проекта вместо вопроса: `CLAUDE.md`, скилы в `.claude/skills/*`, авто-память (`MEMORY.md` и файлы рядом), существующие миграции и `services/*`. Спрашивай только то, чего там нет.
- **Терминология.** Отмечай конфликт языка пользователя с принятым словарём проекта (СТО/parts, роли `sto_owner`/`sto_worker`/…, `sto_company_id`, «заявка» vs «запись» vs «счёт»). Тяни к точному доменному термину.
- **Сценарии-кейсы.** Стресс-тести связи конкретными краевыми случаями (мультитенантность, RLS, инвалидация кэша), чтобы вскрыть размытые границы.
- **Лениво фиксируй решения.** Дорогие к откату/неочевидные решения с реальным трейд-оффом — запиши: ценный долговечный факт → в авто-память (`memory/*.md` + ссылка в `MEMORY.md`); решение по фиче — в `docs/`. Не плоди доки без материала.
