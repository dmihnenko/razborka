# TDD (этот проект)

Тестируем **поведение через публичный интерфейс**, не детали реализации — такие тесты переживают рефакторинг. Стек: Vitest (unit, `npm test`), Playwright (e2e, `npm run e2e`).

## Антипаттерн
Горизонтальная нарезка — написать ВСЕ тесты заранее. Это тесты воображаемого поведения. Вместо неё — **вертикальные срезы**: один тест → одна реализация → повтор.

## Поток
1. **План.** Уточни нужный интерфейс (сигнатура `services/*`/хука/утилиты), приоритезируй проверяемые поведения, заложи тестируемость. Согласуй с пользователем до кода.
2. **Tracer bullet.** Один тест на одно поведение → минимальный код, чтобы он прошёл — доказываем сквозной путь.
3. **Инкремент.** На каждое следующее поведение — один тест за раз, ровно столько кода, чтобы пройти. Не закладывайся на будущие тесты.
4. **Рефактор.** Когда все зелёные — убери дублирование, углуби модули, тесты остаются зелёными.

## Чек-лист цикла
- тест описывает поведение, не реализацию;
- тест использует только публичный интерфейс;
- тест переживёт внутренний рефакторинг;
- кода ровно на текущий тест, без спекулятивных фич.

Для слоя данных мокируй `@/lib/supabase` или тестируй чистые функции `services/*`; не дёргай боевую БД из тестов.
