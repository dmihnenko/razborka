# Handoff (этот проект)

Собери компактный документ-передачу контекста для следующей сессии.

## Куда и как
- Сохрани во **временную папку ОС** (не в рабочее дерево репозитория, чтобы не коммитить случайно).
- **Ссылайся на артефакты по пути/URL**, не дублируй их: миграции БД (путь — см. `CLAUDE.md` проекта), файлы кода, PRD в `docs/*`, факты в памяти проекта, коммиты/диффы.
- **Редактируй чувствительное**: токены/ключи/пароли/PII — не вставлять (помни: токен Supabase — из Credential Manager, в логах не печатается).

## Содержание
- Цель и текущий статус (что сделано, что зелёное, что осталось).
- Затронутые файлы/слои и ключевые решения — кратко, со ссылками, без копирования больших кусков.
- Подводные камни проекта, всплывшие в сессии (RLS-гард definer-RPC, инвалидация react-query, мультитенантность, цикл вендор-чанков).
- Следующий шаг — конкретный.
- **Подходящие скилы** для следующего агента (`tech-lead`, `fullstack-feature`, `supabase-data`, `diagnose`, …).

## Правила
- Цитаты дословно — только в кавычках и короткие (≤125 символов); остальное — пересказ.
- Не повторяй то, что уже есть в PRD/плане/память/коммитах — дай ссылку.
- Долговечные факты лучше класть в авто-память (`MEMORY.md` + файл), а не только в handoff.
